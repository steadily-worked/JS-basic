# 형변환을 주의하자

```js
console.log("1" == 1); // true
console.log(1 == true); // true
console.log(0 == false); // true
```

위와 같은 식은, 사람이 봤을 때는 '느슨한 검사를 하는구나'라고 생각할 수 있지만, (암묵적인)형변환이 일어나고 있는 것이라고 볼 수 있다. 그렇기에 이 검사들이 전부 먹힐 수 있는 것이다.

```js
console.log(11 + " 문자와 결합"); // '11 문자와 결합'
console.log(!!"문자열"); // true
console.log(!!""); // false
// 여기까지 암묵적인 변화

console.log(String(11 + " 문자와 결합")); // '11 문자와 결합'
console.log(Boolean("문자열")); // true
console.log(Boolean("")); // false
console.log(Number("11")); // 11
// 형 변환을 직접 지정해줌으로써 명백한 변화를 하게 됨

console.log(parseInt("9.999", 10)); // 9. 뒤에 들어가는 숫자는 n진수를 나타낸다.
// `parseInt`를 사용했을 때 보통은 10진수를 기대한다. 그렇지만 10을 지정해주지 않는다면 10진수가 기본 값으로 들어가지 않으므로 10진수임을 명확하게 해야 한다.
// 명백한 변화
```

## 결론

- JS를 사용함에 있어서, 사용자가 형변환을 했을 때는 명시적인 형변환이라고 볼 수 있고, 사용자가 아니라 JS Engine이 평가했을 때는 암묵적인 형변환이라고 할 수 있다.
- 따라서 형 변환을 할 때 명시적으로 해줄 필요가 있다. 이를 통해 예측하기 쉬운 타입 변환을 활용하는 것을 추천한다.
